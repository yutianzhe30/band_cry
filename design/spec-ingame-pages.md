# Spec: 游戏内页面设计

当前 MainGame 是一个单屏布局，所有内容都挤在同一个视图里。随着内容增加，需要更清晰的页面结构。

---

## 整体导航结构

游戏内有一条持久的**顶部导航栏（TabBar）**，玩家可以在以下标签页间切换：

```
[ 行动 ]  [ 乐队 ]  [ 日志 ]
```

- **行动**（默认）：每周行动选择，当前已有
- **乐队**：乐队成员与关系管理（新）
- **日志**：历史事件记录（当前 LogPanel 的扩展版）

切换标签页**不消耗 AP，不推进时间**。当有待处理事件时（phase = 'event'/'summary'/'ending'），TabBar 锁定，强制玩家先处理事件。

## 页面一：行动页（Action Page）——现有，微调

### 当前问题
- 属性栏（StatBar）和行动按钮在小屏幕上太拥挤
- 没有对当前周的"情景描述"，缺乏代入感

### 目标布局

```
┌────────────────────────────────────────────────────────────┐
│  [名字]  ·  第N周  ·  18岁  ·  YYYY年M月          [保存]  │  ← header（不变）
├────────────────────────────────────────────────────────────┤
│ [ 行动 ]  [ 乐队 ]  [ 日志 ]                               │  ← TabBar（新）
├──────────┬─────────────────────────────────────────────────┤
│          │                                                   │
│  属性栏  │  场景描述区（中央大图/氛围文字）                 │
│  StatBar │                                                   │
│          │  "这周你感觉还不错，排练室里有新面孔..."         │
│          │                                                   │
├──────────┴─────────────────────────────────────────────────┤
│  ● ● ● ○  4AP          [行动按钮 ×7]          [结束本周→] │  ← 底部 ActionPanel（不变）
└────────────────────────────────────────────────────────────┘
```

**场景描述区**：每周结束时随机从一个小型文字库里选一句"周间氛围描述"，纯文本，不影响游戏逻辑。由一个独立的 `FlavorText.vue` 组件负责展示。

## 页面二：乐队页（Band Page）——新增

### 功能描述

展示玩家当前的**乐队成员**（未来数据驱动，当前占位）和与每个成员的**化学值（chemistry）**。

玩家可在此页对成员执行一些"关系行动"（消耗 AP）。

### 布局

```
┌────────────────────────────────────────────────────────────┐
│  [名字]  ·  第N周  ·  ...                          [保存]  │
├────────────────────────────────────────────────────────────┤
│ [ 行动 ]  [ 乐队 ]  [ 日志 ]                               │
├────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌──────────────────┐  ┌──────────────────┐               │
│  │  🎸 阿磊（吉他） │  │  🥁 小汪（鼓手） │               │
│  │  化学值 ████░░  │  │  化学值 ██░░░░  │               │
│  │  状态：稳定      │  │  状态：不满      │               │
│  │  [约练习 -1AP]   │  │  [沟通 -1AP]     │               │
│  └──────────────────┘  └──────────────────┘               │
│                                                             │
│  ┌──────────────────────────────────────────────────────┐  │
│  │  乐队整体默契：  ████████░░  82                      │  │
│  │  当前成团状态：  排练中                               │  │
│  └──────────────────────────────────────────────────────┘  │
│                                                             │
└────────────────────────────────────────────────────────────┘
```

### 数据扩展

需要在 `GameState` / `PlayerCharacter` 里增加：

```ts
// GameTypes.ts 新增
interface BandMember {
  id: string;
  name: string;
  role: PlayerCharacter['role'];
  chemistry: number; // 0-100，与玩家的关系值
  mood: 'happy' | 'neutral' | 'unhappy';
}

// GameState 新增字段
bandMembers: BandMember[];
```

初始无成员；通过特定事件（`event_recruit_guitarist` 等）加入。

### 关系行动
约练习 / 聚餐 / 沟通 等行动在**乐队页**里显示，效果是提升对应成员 chemistry，消耗 1 AP。

## 事件/结算弹窗（保持不变，覆盖在三个页面之上）

EventCard、SummaryBox、EndingPanel 维持现有的覆盖层设计，不绑定到某个标签页。

---

## 实现优先级

| 优先级 | 内容 | 复杂度 |
|--------|------|--------|
| P0 | TabBar 导航框架 + 日志页（现有内容重组） | 低 |
| P1 | 行动页场景描述区（FlavorText） | 低 |
| P2 | 乐队页 UI（静态占位，无真实数据） | 中 |
| P3 | BandMember 数据结构 + 招募事件 | 高 |
