# Spec: 游戏内页面设计（更新版）

## 整体布局：传统 GAL 风格

主游戏界面采用视觉小说（GAL/VN）经典布局：

```
┌──────────────────────────────────────────────────────────┐
│  [名字] · 第N周 · YYYY年M月                      [保存]  │  ← 顶部 header
├──────────────────────────────────────────────────────────┤
│  [ 行动 ]  [ 乐队 ]  [ 日志 ]                            │  ← TabBar
├──────────────────────────────────────────────────────────┤
│                                                           │
│                   [人物立绘区]                            │  ← 中央场景
│               （目前为方框占位符）                        │
│                                                           │
├──────────────────────────────────────────────────────────┤
│ 技42 魅38 精78 健95 名30 默35 ¥500 [💕35]               │  ← mini 属性条
│──────────────────────────────────────────────────────────│
│ ● ● ● ○  第12周                         [结束本周 →]    │  ← AP + 结束
│──────────────────────────────────────────────────────────│
│ 这周你感觉状态不错，排练室里的空气带着些许兴奋。         │  ← 氛围描述
│──────────────────────────────────────────────────────────│
│ [🎸 练琴]  [🥁 排练]  [🤝 社交]  [📚 学习]  [💼 工作] │  ← 5个行动按钮
└──────────────────────────────────────────────────────────┘
```

事件发生时，VN 文字框替换为事件叙述 + VN 风格选项：

```
│ ──────────────────────────────────────────────────────── │
│ 排练结束后，你在走廊里遇到了一个陌生人——她驻足听了       │
│ 你们排练的最后几段，眼神里有某种真实的专注。             │
│ ──────────────────────────────────────────────────────── │
│  ▸ 走过去搭话                                            │
│  ▸ 点头微笑，各自散去                                    │
└──────────────────────────────────────────────────────────┘
```

## 行动系统：精简为 5 个核心行动

| ID | 名称 | 效果 |
|----|------|------|
| `practice` | 练琴 | 技术+3，精神-2 |
| `rehearse` | 排练 | 乐队默契+5，技术+1，精神-1 |
| `socialize` | 社交 | 魅力+3，默契+3，精神+5，金钱-60 |
| `study` | 学习 | 技术+4，艺术坚持+2，精神-2 |
| `work` | 工作 | 金钱+250，健康-2，精神-1，艺术坚持-1 |

移除了：创作新歌、寻找演出、休息放松（通过事件和自然恢复代替）。

## 属性系统：新增心动值

在原有属性基础上新增 `affection`（心动值），追踪与特定人物的关系发展：

| 属性 | 说明 |
|------|------|
| `affection` | 心动值，0-100。通过NaNa恋爱线事件提升。解锁`nana_romance_path`前不显示。 |

心动值不在普通属性栏中显示，仅在触发`nana_romance_path` flag后在 mini 属性条中以 💕 图标显示。

## 恋爱线：NaNa（贝斯手，乐队成员）

### 人物设定
NaNa是乐队贝斯手，从第一幕起便是队员。与主角的浪漫关系从乐队日常中自然生长，带有"在一起会影响乐队"的内在张力。

### 关系进展

| 阶段 | 事件 ID | 触发条件 | 心动值范围 |
|------|---------|---------|-----------|
| 转折时刻 | `event_nana_moment` | week≥5，随机 | 0→18 |
| 两人单独出去 | `event_nana_hangout` | nana_romance_path + affection>22 | 22→42 |
| 深夜 | `event_nana_night_jam` | nana_hangout + affection>38 | 38→64 |
| 表白 | `event_confess_nana` | nana_night_jam + affection>60 | 60→89 |
| 正式在一起 | `event_relationship_nana` | nana_accepted | 89→100 |

### Flags
- `nana_romance_path` — 恋爱线已解锁
- `nana_hangout` — 两人单独出去过
- `nana_night_jam` — 深夜楼顶
- `confessed_nana` — 已表白（无论结果）
- `nana_accepted` — 表白成功
- `in_relationship` — 正式交往

## 标签页结构（保留）

```
[ 行动 ]  [ 乐队 ]  [ 日志 ]
```

- **行动**（默认）：GAL 风格主界面，VN 文字框 + 5 个行动按钮
- **乐队**：乐队成员卡片（BandPage.vue，当前占位）
- **日志**：按周分组、可折叠的历史记录（LogPage.vue）

事件发生时（phase = 'event'/'summary'/'ending'）TabBar 锁定。
